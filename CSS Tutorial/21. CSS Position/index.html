CSS Layout - position property

이 position 속성은 요소에 사용되는 위치 지정 방법의 유형 (정적, 상대적, 고정, 절대 또는 고정)을 지정합니다.

position 속성

이 position 속성은 요소에 사용되는 위치 지정 방법의 유형을 지정합니다.

5가지 position 값들이 있습니다.

  • static
  • relaitve
  • fixed
  • absolute
  • sticky

그런 다음요소는 top, bottom, left, right 속성을 사용하여 배치됩니다.

그러나 position 속성을 먼저 설정하지 않으면 이러한 속성이 작동하지 않습니다. 또한 위치 값에 따라 다르게 작동합니다.

position static

HTMl 요소는 기본적으로 정적으로 배치됩니다.

정적으로 배치된 요소는 top, bottom, left, right 속성의 영향을 받지 않습니다.

요소에 position: static; 은 특별한 방법으로 배치되지 않습니다. 페이지의 정상적인 흐름에 따라 항상 배치됩니다.

이 div 요소의 position은 static입니다.
    div.static {
      position: static;
      border: 3px solid #73ad21;
    }

position: relative

position: relative는 요소 그 통상의 위치에 관련해 배치됩니다.

상대적으로 배치된 요소의 top, bottom, left, right 속성을 설정하면 해당 요소가 정상 위치에서 벗어나 조정됩니다. 다른 콘텐츠는 요소가 남긴 틈에 맞춰 조정되지 않습니다.

이 div 요소의 position은 relative입니다.
    div.relative {
      position: relative;
      border: 3px solid #73AD21;
      left: 30px;
    }

position: fixed

position: fixed 된 요소 뷰포트를 기준으로 배치되므로 페이지를 스크롤 하더라도 항상 동일한 위치에 유지된다. top, bottom, left, right는 요소의 위치를 결정하는데 사용됩니다.

고정 요소는 일반적으로 위치한 페이지에 간격을 남기지 않습니다.

    div.fixed {
      position: fixed;
      bottom: 0;
      border: 3px solid #73AD21;
      right: 0;
      width: 300px;
    }

position: absolute

position: absolute인 요소는 가장 가까운 position: relative 속성이 존재하는 부모에 상대적으로 위치합니다.

그러나 position: absolute 요소에 위치하는 조상이 없는 경우 문서 본문을 사용하고 페이지 스크롤을 따라 이동합니다.

참고: positioned 이 있는 요소는 position: static을 제외한 모든 것을 나타내는 것이다.

position: sticky

position: sticky인 요소는 사용자의 스크롤 위치를 기반으로 배치됩니다.

skicty 요소가 스크롤 위치에 따라 상대와 고정 사이를 전환한다. 지정된 offset 위치가 뷰포트에서 충족될 떄까지 상대위치로 배치되며 그런다음 position을 fixed 시킨다.

참고: IE, Edge 15 이전 버전은 position:sticky를 지원하지 않습니다. Safari에는 -webkit- 접두사가 필요합니다. 또한 top, bottom, left, right 중 하나이상 지정해야지 작동합니다.

겹치는 요소들

요소가 배치되면 다른 요소와 겹칠 수 있습니다.

이 z-index속성은 요소의 스택 순서를 지정합니다. (요소는 다른 요소의 앞 또는 뒤에 있어야한다.)

요소는 양수 또는 음수 값을 가질수 있습니다.

이것은 제목이다.

이미지의 z-index가 -1이므로 텍스트 뒤에 배치됩니다.
